java中的语言分隔符和标志符

Map
分号（;）：语句的分割，表示一句话结束
花括号（{}）：表示一个代码块，是一个整体，花括号要成对使用
方括号（[ ]）：定义数组和访问数组元素时使用
圆括号（( )）：使用很广泛
圆点（.）：类和对象访问它的成员时使用
空格（ ）：将一整条语句分割为几段

注意：必须都是半角下的英文符号
————————————————————————————
标志符：我们把这种增强程序阅读性而自定义的名称，称为标志符
标志符命名规则：
1、由字母、数字、下划线组成，但不能以数字开头（注：此处的字母可以是中文、日文等）
2、大小写敏感
3、不得使用java中的关键字与保留字
4、不用java中内置的类名作为自己的类名

注意问题：类名首字母规范是大写

关键字与保留字

Map
关键字：在编程语言中有一些事先定义的，有着特殊含义的单词
保留字：和关键词一样都是编程语言事先定义好的，只是说现在暂时没有特殊的用途，但说不定以后某天会突然被赋予意义和被使用到，因此被保留下来的单词，goto和const就是java的保留字

注意：java无sizeof（既不是关键字也不是保留字）,goto,const关键字
           关键字都是小写

java的三大注释

Map
1、单行注释：// ; //注释信息，从//开始道本行结束的所有字符会被编译器忽略
2、多行注释：/* */; /*注释信息*/之间的所有字符会被编译器忽略
3、文档注释：/**  */;/**注释信息*/和多行注释一样，除此以外还可以专门生成文档信息API

注意：多行注释和文档注释之间

基本语法

Map
java的基本语法：
1、java语言严格区分大小写
2、在一个java的源文件中可以定义多个类，但是只有一个类可以被定义为public，且public的类名必须和文件名相同
3、一个源文件中包含N个类时，成功编译后悔生成N份字节码文件，每个类都会生成一份单独的class文件
4、一个类必须运行，则必须拥有主方法，因为主方法时程序的入口

编程建议：
1、一个java源文件只定义一个类，不同的类使用不同的源文件定义
2、将每个源文件中单独定义的类都定义为public的
3、保持java源文件的主文件名与源文件中的类名一致

JDK-JRE-JVM概述

Map
JDK：java开发工具，包含开发java程序的人所有工具，JDK中包含JRE
JRE：java运行环境，如果要运行java程序，就需要JRE的支持，JRE里包含JVM
JVM：运行所有java程序的虚拟计算机，好比是街机优秀的模拟器
           JVM用于读取并处理编译过的与平台无关的字节码（class）文件，从而实现java的可移                       
           植性

java的跨平台性原理

Map
高级语言：
        代码中：源代码
        存放代码的文件：源文件
与平台无关的文件：字节码文件，class文件
java编译的工具；javac(和任何的平台无关，不能直接运行在mac、win、linux)
                               Java虚拟机(JVM)：可以识别字节码文件(虚拟机就是模拟器)
启动JVM的工具：java
class文件只能被JVM识别

Java跨平台的原理：1、编译之后会生成与平台无关的字节码文件
                                 2、得来自不同平台的虚拟机（JVM）
                                 JVM不是跨平台的

sun公司和java平台

Map
1982年，Sun Microsystems公司诞生于美国斯坦福大学校园。Sun公司1986年上市，在NASDAQ的标识为SUNW。 2007年改为JAVA
2009年4月20日,美国数据软件巨头甲骨文公司(Oracle)宣布以74亿元收购SUN公司,从此Java也有干爹了,在这个拼爹的时代java的发展前景毋庸置疑


1999年6月，SUN公司发布Java的三个版本：标准版、企业版和微型版；2005年J2SE,J2EE,J2ME分别改名为JavaSE JavaEE JavaME;随着J2EE的发布,Java正式迎来了自己的"春天"
Java SE(Java Platform Standard Edition):Java标准平台,他允许开发和部署在桌面,服务器,嵌入式环境和实时环境中使用的Java应用程序;JavaSE能为JavaEE和JavaME提供开发基础.


Java EE(Java Platform Enterprise Edition):Java企业平台,可帮助开发和部署可移植,健壮,可伸缩且安全的服务器端Java应用程序,,如企业的应用系统ERP等.JavaEE是在JavaSE的基础上构建的,他提供web服务,组件模型,管理和通信API,可以用来实现企业级的面向服务体系结构(SDA)和Web等应用程序.
Java ME(Java Platform Micro Edition):Java微型平台,也叫K-JAVA.用来开发移动设备和嵌入是设备(比如手机,PDA,电视机顶盒和打印机等.随着Android的迅猛发展,javaME现在使用的越来越少了,Android会逐渐取代JavaME的市场份额
以Java作为开发语言的其他设备有很多,如:Android智能设备.
学习Java,我们都是从javaSE开始的,只有学号JavaSE才能很好的掌握JavaEE和Android的知识.对于Java的就业,很少有人从事单纯的JavaSE开发的,这不是Java的强势支出,我们应该面向市场广阔的JavaEE和Android方向
